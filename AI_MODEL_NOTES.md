# 🤖 Финальная логика работы генерации аватаров — A-to-A

## Текущая конфигурация (обновлено 2026-04-29)

**Модель:** `gemini-2.5-flash`  
**Способ вызова:** Прямой REST API (через `fetch`)  
**Файл реализации:** `src/app/api/generate-avatars/route.ts`

---

## Что было исправлено и почему

### 1. Отказ от `responseSchema` (Structured Output)
**Проблема:** При использовании сложной вложенной схемы JSON (как в нашем исследовании аватара), модель `gemini-2.5-flash` уходила в глубокий "Thinking" режим. Генерация одного аватара занимала **более 4 минут**, что приводило к таймаутам сервера Next.js.
**Решение:** Схема удалена из параметров запроса. Промпты в `prompts.ts` сами по себе очень жесткие и детальные — модель прекрасно выдает валидный JSON и без принудительной схемы.
**Результат:** Время генерации сократилось с 4+ минут до **40-60 секунд**.

### 2. Удаление лимита `maxOutputTokens`
**Проблема:** Ранее стоял лимит 4096 или 8192 токена. Для глубокого исследования аватара этого не хватало. Как только модель достигала лимита, она обрывала текст на середине слова, ломая структуру JSON. Это вызывало ошибку `Unterminated string in JSON`.
**Решение:** Лимит токенов удален. Теперь модель завершает генерацию объекта естественным образом.
**Результат:** Исчезли ошибки битого JSON.

### 3. Переход на прямой REST API (Fetch)
**Проблема:** SDK `@google/generative-ai` версии 0.24.x не позволял гибко управлять некоторыми параметрами и иногда игнорировал настройки таймаутов.
**Решение:** Код переписан на стандартный `fetch`. Это позволило убрать лишние промежуточные таймауты (AbortSignal), которые обрывали соединение на 2-й минуте.
**Результат:** Стабильное соединение до 5 минут (лимит `maxDuration = 300` в Next.js).

## Текущая логика обработки ответа

1. Система отправляет полный промпт из `prompts.ts`.
2. Получает текстовый ответ от Gemini.
3. Использует регулярное выражение `text.match(/\{[\s\S]*\}/)` для поиска JSON-объекта в тексте (на случай, если модель добавит лишние пояснения или markdown-разметку ```json).
4. Парсит найденный объект через `JSON.parse()`.

## Рекомендации по поддержке

- **Не возвращайте `responseSchema`** для исследования аватара, пока Google радикально не ускорит этот режим для вложенных массивов.
- **Промпт в `prompts.ts` является святыней**: качество аватара на 100% зависит от него. Если нужно добавить данных в аватар — добавляйте их в текст промпта и в `required` список в API роуте для контроля.
- **Модель `gemini-2.5-flash`** на данный момент является оптимальной по соотношению "глубина психологии / скорость".

## Как проверить работоспособность (через терминал)

```bash
source .env.local && curl -s -X POST \
  "http://localhost:5177/api/generate-avatars?step=research&segment=Test" \
  -H 'Content-Type: application/json' \
  -d '{"productName":"Тест"}'
```
*Если ответ пришел за 40-60 секунд с полным объектом — всё работает идеально.*
