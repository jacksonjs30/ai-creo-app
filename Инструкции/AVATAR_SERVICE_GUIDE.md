# 📘 Полное руководство по системе AI Avatar Generation (A-to-A)

Данный документ описывает архитектуру, логику работы и технические решения, обеспечивающие высокую скорость и качество генерации маркетинговых аватаров.

---

## 🏗 1. Общая архитектура

Система построена на трех уровнях:

1.  **Интерфейс (Frontend):** `src/app/project/new/page.tsx`
    - Управляет состоянием (Step-by-Step).
    - Выполняет параллельные запросы к API для экономии времени.
2.  **Серверный роут (API):** `src/app/api/generate-avatars/route.ts`
    - Проксирует запросы к Google Gemini.
    - Обрабатывает логику "Identify" (поиск сегментов) и "Research" (глубокое исследование).
3.  **База знаний (Prompts):** `src/lib/prompts.ts`
    - Содержит экспертную маркетинговую методологию (JTBD, CJM, Психографика).

---

## ⚙️ 2. Логика работы (Step-by-Step)

### Шаг 1: Идентификация сегментов (`step=identify`)
- **Вход:** Данные брифа (продукт, описание, гео).
- **Процесс:** Если пользователь ввел сегменты вручную — этот шаг пропускается. Если нет — ИИ анализирует бриф и предлагает 10 наиболее релевантных сегментов ЦА.
- **Оптимизация:** Используется облегченная схема JSON для мгновенного ответа.

### Шаг 2: Выбор и фильтрация
- Пользователь выбирает до **4-х сегментов** (лимит увеличен с 3 до 4 по требованию).
- Список выбранных сегментов фиксируется в состоянии фронтенда. При ошибках генерации система возвращает пользователя к списку 10 сегментов без потери данных.

### Шаг 3: Глубокое исследование (`step=research`)
- **Параллелизм:** Фронтенд запускает запросы для каждого выбранного сегмента ОДНОВРЕМЕННО с интервалом 500мс.
- **Логика:** Модель получает имя сегмента и бриф. Она должна сгенерировать:
  - Психологический портрет (6-8 предложений).
  - 5-7 задач JTBD с прямыми цитатами.
  - 5-7 болей, страхов и симптомов.
  - Подробный CJM (сценарий "До").

## 🚀 3. Технические решения для скорости и стабильности

Чтобы добиться генерации за ~50 сек вместо 4-5 минут, были внедрены следующие решения:

### ⚡️ Отказ от Structured Output (`responseSchema`)
- **Почему:** Google Gemini 2.5 Flash при получении сложной вложенной схемы JSON переходит в режим "Thinking" (внутренние рассуждения). Это добавляет 3 минуты задержки на каждый запрос.
- **Решение:** Мы передаем структуру JSON прямо в тексте промпта. Модель выполняет её идеально без принудительной схемы.
- **Результат:** Скорость выросла в 5 раз.

### 📦 Удаление лимитов токенов
- **Почему:** Лимит в 8192 токена обрезал JSON-ответ на середине, если аватар получался слишком детальным.
- **Решение:** Параметр `maxOutputTokens` удален. Модель сама решает, когда закрыть JSON-объект.
- **Результат:** 100% валидность JSON даже для очень длинных исследований.

### 🛠 Прямой вызов REST API (Fetch)
- **Почему:** Стандартный SDK (`@google/generative-ai`) накладывает свои ограничения и скрывает детали реализации (например, невозможность отключить Thinking в некоторых версиях).
- **Решение:** Использование нативного `fetch` к эндпоинту `v1beta`.
- **Преимущество:** Полный контроль над таймаутами и заголовками.

---

## 🛡 4. Надежность и отказоустойчивость

- **Таймауты:** Сервер настроен на `maxDuration = 300` (5 минут). Этого достаточно даже для самых длинных генераций.
- **Обработка ошибок:**
  - Если один аватар из 4-х упал, фронтенд помечает его ошибкой и пробует еще раз (до 3 попыток).
  - В роуте реализован RegExp-парсинг: `text.match(/\{[\s\S]*\}/)`. Это позволяет извлечь чистый JSON, даже если модель добавила лишний текст до или после объекта.
- **Сохранение состояния:** Исходные 10 сегментов (`foundSegments`) не затираются при переходе к исследованию. Если пользователь захочет сменить выбор или генерация упадет — он вернется к исходному списку выбора.

## 📈 5. Варианты развития и настройки

### Изменение модели
Если требуется еще более глубокое качество (ценой скорости), в `route.ts` можно сменить `gemini-2.5-flash` на `gemini-1.5-pro` (если она доступна по вашему тарифу).

### Настройка "Глубины"
Чтобы аватары стали еще детальнее, нужно изменять `src/lib/prompts.ts`. 
*Внимание:* Увеличение количества требуемых пунктов (например, до 15 болей) может снова приблизить нас к лимиту времени в 5 минут.

### География и Язык
Система автоматически подстраивается под язык брифа. Промпты написаны на русском, но модель может генерировать на любом языке, если бриф подан на нем (однако методология жестко зашита на русском для точности терминов JTBD).
